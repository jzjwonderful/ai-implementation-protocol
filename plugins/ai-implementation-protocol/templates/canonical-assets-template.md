# 正典资产台账 · 活文档

> **用途**：登记"该被复用的正典构件"——共享工具、脚手架、验证机制、测试夹具。开发**造新前先查这里**（命中就复用或扩展，不另起并行实现）。防止越迭代越堆积。
> **维护铁律**：新增/退役正典构件时**同次更新**本表。本表只列"钦定该复用的"；全量符号发现用项目的索引工具（codegraph/nexus/grep）——**手动索引查前先刷新**。

## 怎么用
- **开工前（设计阶段）**：查本表看本轮能复用/扩展谁、哪个旧构件该退役。
- **实现时（复用优先闸门）**：造任何新工具/机制/脚手架前，先刷新并查索引 + grep 已有 → 命中就复用/扩展；**造新须在变更摘要写明"为什么已有的不适用"**。没有理由 = 不许造新。
- **验收时（反堆积闸门）**：新增正典构件是否登记进本表；有没有留下被取代却没删的旧机制（Strangler：迁移消费者 → 同次删旧）。
- **撞见无关问题**：登记 `findings.md`，别就地修跑偏。
