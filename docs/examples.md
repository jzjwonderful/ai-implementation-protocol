# Examples

## C++ / Qt Project

Use AIP for:

- feature specs
- refactor plans
- file scope control
- verification command records

Good fit:

- desktop applications
- embedded tooling
- protocol-heavy systems

## Python Service

Use AIP for:

- API change tracking
- migration handoff
- incident follow-up
- multi-agent implementation

## Frontend Project

Use AIP for:

- design-spec alignment
- route/component scope tracking
- browser verification notes

## Common Pattern

Across all examples, the same structure remains stable:

- one runtime pointer
- one feature directory
- one machine-readable task board
- one enforced handoff format
